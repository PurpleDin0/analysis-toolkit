About This Toolkit

▼

Purpose

The NDU Analyst Toolkit provides structured prompts for NotebookLM to help Senior Defense University students systematically analyze course readings using the "Gutting a Book" methodology. Each prompt is designed to produce rigorous, skeptical, executive-level analysis filtered through course-specific analytical lenses.

Design Principles

  • Syllabus First: Every analysis begins by extracting Learning Objectives and Issues for Consideration from the syllabus, then filters all output through those objectives.
  • 14-Header Framework: A standardized analytical template covering Document Identity, Content & Argument, Critical Evaluation, and Significance & Application.
  • Course-Specific Lenses: Each course applies persistent analytical frameworks (e.g., SAR uses acquisition process alignment; MSW uses Ends-Ways-Means; EMSO uses Triple Helix).
  • Batched Workflow: Readings are analyzed in batches of 3-5 to ensure comprehensive, untruncated output for each document.

Batched Workflow (v2.2)

The v2.2 workflow processes readings in manageable batches to prevent output truncation:

  1. Step 0 — Custom Role: Configure the AI persona in NotebookLM Chat Settings with course-specific analytical lenses.
  2. Step 1 — Preliminary: Inventory all readings, verify document integrity, and get batching recommendations.
  3. Step 2 — Gutting Batches: Analyze 3-5 readings per batch through the full 14-header framework. Repeat for each batch.
  4. Step 3 — Synthesis: Generate cross-reading synthesis with course-specific Framework Check and Issues for Consideration.
  5. Step 4 — Media (Optional): Generate Podcast, Video, or Slides overviews using NotebookLM Studio.
